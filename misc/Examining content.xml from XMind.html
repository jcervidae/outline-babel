Examining content.xml from XMind

The XML declaration can't be done properly by lxml at present because it doesn't support standalone. I don't know if XMIND would care but lets write it by hand to the file to be safe as the first line: <?xml version="1.0" encoding="UTF-8" standalone="no"?> ID's appear to be random strings of 27 lower-case letters and numbers with the first character always being a number. It is assumed these must be unique. Encased in sheet element which has id property. There are topics with id property. Even the main topic is a topic. <xmap-content xmlns="urn:xmind:xmap:xmlns:content:2.0" xmlns:fo="http://www.w3.org/1999/XSL/Format" xmlns:svg="http://www.w3.org/2000/svg" xmlns:xhtml="http://www.w3.org/1999/xhtml" xmlns:xlink="http://www.w3.org/1999/xlink" version="2.0"> <sheet id="XXX"> <topic id="XX"> <title>The Middle Block Title/title> <children> <topics type="attached"> REST OF DOCUMENT GOES HERE </topics> </children> </topic> <title>The Sheet Title</title>< </sheet> </xmap-content> Rest of document sits in the <children> element of the project. Branches with children formatted like:. <topic id="XXX> <title>A topic with sub-topics</title> <children> <topics type="attached"> </topics> </children> </topic> And the end branches (leaves?) with no children sit between the <topics> element of branches. They Look like this: <topic id="XX"> <title>I am a leaf</title> </topic>
